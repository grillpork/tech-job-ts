# 🔌 เอกสารสรุป API แต่ละส่วน (API Reference)
เอกสารชุดนี้สรุปว่า API แต่ละจุดทำหน้าที่อะไร เพื่อใช้ในการอธิบายระบบหลังบ้าน

---

## 1. กลุ่มจัดการงาน (Jobs API)
ส่วนที่สำคัญที่สุดของระบบ จัดการวงจรชีวิตของใบงาน (Job Lifecycle)
*   **`/api/jobs`**: 
    *   `GET`: ดึงรายการงานทั้งหมดพร้อมข้อมูลความสัมพันธ์ (ช่างที่รับผิดชอบ)
    *   `POST`: สร้างใบงานใหม่ บันทึกชื่อคนสร้าง และสถานะเริ่มต้นเป็น 'pending'
*   **`/api/jobs/[id]`**: 
    *   `PUT`: อัปเดตข้อมูลงาน เช่น เปลี่ยนช่าง, เปลี่ยนสถานะ, หรือเพิ่มโน้ต
    *   `DELETE`: ลบใบงานออกจากระบบ (เฉพาะ Admin)
*   **`/api/jobs/completion`**: 
    *   `POST`: ช่างส่งคำขอปิดงาน พร้อมแนบลายเซ็นดิจิทัล (Base64) และรูปภาพก่อน/หลังงาน

---

## 2. กลุ่มจัดการพัสดุ (Inventory API)
จัดการการเบิกจ่ายและสต็อกอุปกรณ์
*   **`/api/inventory`**: 
    *   `GET`: เช็คสต็อกอุปกรณ์ทั้งหมด ดูจำนวนคงเหลือ และสถานะ
    *   `POST`: เพิ่มอุปกรณ์ใหม่เข้าคลัง (ใส่รูปภาพและระบุจุดเก็บ)
*   **`/api/inventory/requests`**: 
    *   `POST`: สร้างคำขอเบิกพัสดุสำหรับงานนั้นๆ ระบบจะตัดสต็อกแบบชั่วคราว
*   **`/api/inventory/requests/[id]`**: 
    *   `PUT`: ผู้จัดการหรือหัวหน้าช่างกด 'อนุมัติ' หรือ 'ปฏิเสธ' การเบิก

---

## 3. กลุ่มผู้ใช้งานและการเข้าถึง (Users & Auth API)
ควบคุมการเข้าสู่ระบบและข้อมูลพนักงาน
*   **`/api/auth/[...nextauth]`**: ระบบจัดการ Token และตรวจสอบ Session (ใช้ Login/Logout)
*   **`/api/users`**: ดึงรายชื่อพนักงานทั้งหมดแยกตามแผนก (ไฟฟ้า, ประปา, เทคนิค)
*   **`/api/register`**: ระบบลงทะเบียนพนักงานใหม่ และเข้ารหัส Password ด้วย Bcrypt

---

## 4. กลุ่มตรวจสอบและสถิติ (Monitoring & Analytics API)
ใช้สำหรับดึงข้อมูลมาแสดงเป็น Graph และเก็บประวัติย้อนหลัง
*   **`/api/audit-logs`**: สืบค้นประวัติการทำงาน ใครทำอะไรกับงานชิ้นไหน (ใช้ตรวจสอบความโปร่งใส)
*   **`/api/dashboard/stats`**: คำนวณสถิติภาพรวม เช่น งานที่เสร็จแล้วกี่ %, พัสดุที่กำลังจะหมดกี่รายการ
*   **`/api/analytics/summary`**: สรุปข้อมูลสำหรับหน้า Dashboard ของผู้บริหาร

---

## 5. กลุ่มแจ้งปัญหาและระบบเสริม (Utility API)
*   **`/api/reports`**: จัดการรายงานปัญหา (Bug/Incident) จากพนักงานหน้างาน
*   **`/api/upload`**: ตัวกลางจัดการอัปโหลดไฟล์รูปภาพขึ้นระบบ
*   **`/api/docs`**: ระบบ Swagger UI แสดงเอกสาร API ทั้งหมดแบบ Interactive

---

### 💡 จุดเด่นที่ควรอธิบายให้อาจารย์ฟัง:
1.  **RESTful Standard:** เราออกแบบ API ตามมาตรฐาน REST ทำให้ขยายระบบได้ง่าย
2.  **Atomicity:** การประมวลผลข้อมูลใน API มั่นใจได้ว่าข้อมูลจะไม่ผิดเพี้ยน (เช่น ถ้าเบิกพัสดุไม่ผ่าน สต็อกจะไม่ถูกตัด)
3.  **JSON Persistence:** ในหลายๆ API เราใช้การเก็บข้อมูลโครงสร้างซับซ้อนในรูปแบบ JSON เพื่อลด latency ในการ Join ตารางครับ
